接收http请求

后台使用线程池进行处理, 每次都领取一个任务进行执行, 领取任务执行以后, 返回状态信息, 前端变为waiting状态, 否则则变成error(代表任务上传失败)

如果任务能正常执行, 则前端则继续在wait状态, 执行成功以后, 会返回执行的结果, 任务正常结束是ending, 如果任务非正常结束(比如线程崩坏, 或者是计算问题), 则返回给状态"mistake"


具体的模型预测

模型预测部分需要加载为python脚本, 削减长度, 产生图片等等工作全都交给python脚本进行处理

线程的任务就是接受序列或者文件, 然后调用脚本进行处理

需要单独设计一个文件上传模块, 用来管理模块中的上传文件, 删除文件, 维护文件信息的功能, 相当于文件目录了()

### 文件操作模块(主要面向csv,fastq文件, 以及motif图片)
> 
> 上传: 主要需要上传csv, fastq文件, 以http格式进行传递, 传递为二进制文件, 并且保存在后端固定文件夹中, 返回文件的类型, 以及名称
>
> 发送: 主要用来发送motif图片, 以http格式进行传递, 生成的motif转化为二进制文件, 进行传递, 如果保证传递成功, 则删除对应文件
>
> 删除: 根据文件名称删除对应文件
>
> 日志: 删除, 上传, 生成文件, 都需要用日志记录时间戳, 以及相关内容

### http处理模块(用来处理http请求的相关信息)
> 
> 解析内容信息, 并且保存在map里面
> 
> body中的内容保存在json对象中
